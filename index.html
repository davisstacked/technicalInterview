var, let, and const

Var is function scoped. let and const are block scope. Block scope means anything with curly braces let and const can't escape to higher scope. so if statements, loops, anything with curly braces

Var causes security risks and it can cause confusion

Object oriented vs functional programming

Functional Programming

the most important concept in functional programming is the concept of Pure Functions. This means the output of your function should only depend on its inputs.

We can make it an inpure function by manipulating the numbers directly. This produces a side effect and functional code should produce no side effects.

another core part of immutable data is that it is stateless - when data is created it is never mutated.

Object oriented

Object oriented, or imperative, gives you a clear set of directions to follow to get a cake from recipe to final state. Where as functional programming, declarative, explains the state and logic involved but has no opinion on the actual control flow.

Inheritance vs. Composition for code reusability.
with INHERITANCE you start with a larger base class then have child classes inherit all of this functionality and override or extend it with whatever functionality they need
with COMPOSITION breaks apart the interfaces and logic into a lot of small pieces then builds up larger functions or objects by combining these pieces together.
